# i18n Guide

## Files

- `i18n/de.json`
- `i18n/en.json`

## Key Usage Patterns

- `data-i18n="key"` for text nodes
- `data-i18n-placeholder="key"` for placeholders
- `data-i18n-aria-label="key"` for aria labels
- `data-i18n-title="key"` for title attributes
- `t("key")` in JavaScript

## Conventions

- Use lower snake_case keys
- Add keys in DE and EN in the same change
- Keep placeholders identical between languages (`{name}`, `{count}` etc.)
- Avoid hardcoded user-visible strings in HTML/JS

## QA Modes (Dev)

- Pseudolocalization:
  - URL: `?pseudoloc=1`
  - localStorage: `fp_pseudoloc=1`
- Language QA key mode:
  - URL: `?i18nqa=1`
  - localStorage: `fp_i18n_qa=1`

## Audit

Run:

`npm run lint:i18n`

Outputs:

- `reports/i18n-report.json`
- `reports/i18n-report.md`

The audit fails (exit code != 0) on:

- Missing keys in DE/EN
- Placeholder mismatch between DE/EN
